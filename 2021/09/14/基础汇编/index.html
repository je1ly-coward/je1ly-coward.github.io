基础汇编

Created2021-09-14|Updated2021-10-26
|Post View:

汇编语言程序员处理的是物理级数据，因此他们必须善于检查内存和寄存器。通常，二进制数被用于描述计算机内存的内容；有时也使用十进制和十六进制数。所以必须熟练掌握数字格式，以便快速地进行数字的格式转换。
每一种数制格式或系统，都有一个基数（base），也就是可以分配给单一数字的最大符号数。下表给岀了数制系统内可能的数字，这些系统是硬件和软件手册中最常使用的。

二进制（bit）整数
从 LSB 开始，无符号二进制整数中的每一个位代表的是 2 的加 1 次幂。下图展示的是对一个 8 位的二进制数来说，2 的幂是如何从右到左增加的。

无符号二进制整数到十进制数的转换
对于一个包含 n 个数字的无符号二进制整数来说，加权位记数法（weighted positional notation）提供了一种简便的方法来计算其十进制值：dec = ( Dn-1 x 2n-1 ) + ( Dn-2 x 2n-2 ) +…+ ( D1 x 21 )+ ( D0 x 20 )

无符号十进制整数到二进制数的转换
将无符号十进制整数转换为二进制，方法是不断将这个整数除以 2，并将每个余数记录为一个二进制数字。

二进制加法运算
两个二进制整数相加时，是位对位处理的，从最低的一对位（右边）开始，依序将每一对位进行加法运算。
1 与 1 相加的结果是二进制的 10（等于十进制的 2）。多出来的数字向更高位产生一个进位。
二进制减法运算
如果采用与十进制减法相同的方法，那么从一个较大的二进制数中减去一个较小的无符号二进制数就很容易了。

无符号十六进制数到十进制的转换
十六进制数中，每一个数字位都代表了 16 的幂。这有助于计算一个十六进制整数的十进制值。假设用下标来对一个包含 4 个数字的十六进制数编号 D3D2D1D0
dec = (Dn-1 X 16n-1)+ (Dn-2 x 16n-2)+…+ (D1 X 161)+ (D0 x 160)

字节简介
无符号整数可能的取值范围：
无符号字节 0 到 28-1 ; 无符号四字 0 到 264-1 ; 无符号字 0 到 216-1 ;无符号八字 0 到 2128-1 ; 无符号双字 0 到 232-1 32。

补码
正数则是本身，负数将一个二进制整数按位取反（求补）再加 1，就形成了它的补码。

布尔代数
非（NOT）：标记为 ¬ 或 ~ 或 ‘
与（AND）：标记为^或 ·
或（OR）：标记为 ∨ 或 +
NOT
NOT 运算符将布尔值取反。用数学符号书写为 ¬X，其中，X是一个变量（或表达式），其值为真（T）或假（F）。
AND
布尔运算符 AND 需要两个操作数，用符号表示为 X ^ Y，
当两个输入都是真时，输出才为真。
OR
布尔运算符 OR 需要两个操作数，用符号表示为 X∨Y。
当两个输入都是假时，输出才为假。NOT 运算符具有最高优先级，然后是 AND 和 OR 运算符。
